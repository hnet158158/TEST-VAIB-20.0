# vaib/00-intent/intent.md
## Project: mini_db (VAIB Stress-Test Benchmark)
## Date: 2026-03-16

### 1. Current Reality
Мультиагентный пайплайн VAIB создан для автоматизации разработки сложных программных систем. Однако возможности агентов по:
- Самостоятельной декомпозиции сложных алгоритмических задач
- Удержанию контрактов между модулями
- Разрешению архитектурных противоречий без участия человека

...остаются непроверенными на реальном инженерном стресс-тесте. Требуется компактный, но алгоритмически насыщенный проект для валидации качества оркестрации.

### 2. Core Intent
**Доказать, что VAIB-агенты способны самостоятельно реализовать систему с нетривиальной архитектурой** (парсер, AST, storage engine, execution layer) и пройти все критические точки:
1. Честный парсинг вложенных скобок AND/OR без eval/regex
2. Атомарность UPDATE (rollback при ошибке)
3. Graceful error handling без Python Traceback в REPL

Проект mini_db — это инструмент валидации VAIB, не продакшен-СУБД.

### 3. Blind Spots & Pivot

**Risks:**
- **Scope Creep**: Риск добавления фич сверх ТЗ (JOIN, транзакции, etc.) — за рамками бенчмарка
- **Over-engineering**: Агенты могут построить избыточную архитектуру вместо минимально достаточной
- **Integration Hell**: Несостыковки между парсером и executor при отсутствии чётких контрактов
- **Test Coverage Gaps**: Тесты сами по себе должны покрывать краевые случаи

**Pivot:**
Минимальный путь — строго следовать ТЗ без расширения scope. Система состоит из:
- Lexer/Parser (recursive descent, без regex)
- AST nodes
- Storage engine (in-memory tables, indexes)
- Execution engine с rollback механизмом
- REPL interface

**Отвергнутые альтернативы:**
- SQLite wrapper — не проверяет парсер и архитектуру
- Использование pyparsing/lark — нарушает "чистый Python без зависимостей"

### 4. Success Metric
Бенчмарк считается пройденным при:

**Количественные метрики:**
1. Все тесты в Test Suite проходят (0 failures)
2. 3 критических checkpoint'а покрыты тестами:
   - `[TEST]` Парсер строит корректное AST для `(col1 > 10 OR col2 = 'test') AND col3 != true`
   - `[TEST]` UPDATE нескольких строк откатывается полностью при нарушении UNIQUE
   - `[TEST]` REPL выводит "Syntax error: ..." вместо Python Traceback

**Качественные метрики:**
- Код разбит на логические модули (parser, ast, storage, executor, repl)
- Агенты не обращались за помощью человека для разрешения архитектурных конфликтов
- VAIB pipeline отработал от Visionary до Skeptic без ручных корректировок
